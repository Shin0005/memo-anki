# Deck機能 E2E試験項目書 (E2E Test Plan)

## 1. 試験実施方針

- [cite_start]**目的**: 層をまたいだロジックの結合確認として、各CRUD操作がDBに正しく反映され、期待通りのHTTPステータスが返却されることを担保する [cite: 1]。
- **範囲**: APIエンドポイント (`/decks` 等) を通じた一連のリクエスト・レスポンスおよびDBの状態。
- **手法**: Vitest + Supertest + 実際のテスト用データベース。
- [cite_start]**特記事項**: 異常系は全ての例外を網羅せず、各メソッドにつき代表的な例外が一つ正しく投げられるかを確認する [cite: 1]。

---

## 2. E2E試験項目 (正常系)

| メソッド   | テストケース     | 確認事項 (Assertion)                                                                    | 優先理由             |
| :--------- | :--------------- | :-------------------------------------------------------------------------------------- | :------------------- |
| **POST**   | デッキ作成成功   | [cite_start]201が返り、DBの件数が1件増加、返却値とDBの値が一致すること [cite: 3]        | 基本フローの担保     |
| **GET**    | 全件取得成功     | [cite_start]200が返り、ログイン中のユーザーが所有する全デッキが取得できること [cite: 3] | データ取得の正確性   |
| **GET**    | 他者データの分離 | [cite_start]取得したデータの中に、別ユーザーのデッキが混じっていないこと [cite: 3]      | 認可・データ分離     |
| **PUT**    | デッキ更新成功   | [cite_start]200が返り、対象レコードが更新され、他のレコードに影響がないこと [cite: 3]   | 更新ロジックの整合性 |
| **DELETE** | デッキ削除成功   | [cite_start]204が返り、対象のレコードがDBから完全に削除されていること [cite: 3]         | 削除処理の完結       |

## 3. E2E試験項目 (異常系・その他)

| メソッド   | テストケース   | 確認事項 (Assertion)                                                                      | 備考                 |
| :--------- | :------------- | :---------------------------------------------------------------------------------------- | :------------------- |
| **共通**   | 認証エラー     | [cite_start]JWTなし、または不正なJWTでアクセスした場合に401エラーとなること [cite: 1]     | ガード機能の確認     |
| **共通**   | バリデーション | [cite_start]不正な値（空のnameや不正な形式のID）を入力すると400エラーが返ること [cite: 3] | 入力値チェック       |
| **POST**   | 名称重複エラー | [cite_start]既に存在する`deckname`で作成を試みた際、409 (Conflict) が返ること [cite: 3]   | ビジネス制約         |
| **PUT**    | 他者デッキ更新 | [cite_start]他人の`deckId`を指定して更新を試みた際、404 (NotFound) が返ること [cite: 3]   | 認可(他人の操作不可) |
| **PUT**    | 名称重複エラー | [cite_start]更新後の名前が他ユーザーのデッキ名と重複する場合、409が返ること [cite: 3]     | 重複チェックロジック |
| **DELETE** | 他者デッキ削除 | [cite_start]他人の`deckId`を指定して削除を試みた際、404が返り、削除されないこと [cite: 3] | 認可の担保           |
