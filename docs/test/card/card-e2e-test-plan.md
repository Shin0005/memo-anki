## 1. 試験実施方針

- **目的**: APIエンドポイントからDB操作までの一連の流れが、仕様（認可・ビジネスロジック）通りに動作することを確認する。
- **範囲**: `CardController` (API) ～ `CardRepository` (DB)
- **手法**: Vitest + TestAgent (Supertest等) + 疎通確認用テストDB

---

## 2. 正常系シナリオ (CRUD)

| シナリオ   | テストケース         | 確認事項 (Assertion)                                                                    | 備考    |
| :--------- | :------------------- | :-------------------------------------------------------------------------------------- | :------ |
| **Create** | NOTE形式の作成       | Status 201。DBにレコードが作成され、`question`/`answer` が強制的に `null` であること。  | type: 0 |
|            | QUIZ形式の作成       | Status 201。DBにレコードが作成され、`content` が強制的に `null` であること。            | type: 1 |
| **Get**    | 自分のカード一覧取得 | Status 200。作成したカードが含まれ、レスポンスの各フィールドがDBの値と一致すること。    |         |
|            | データの分離確認     | 他のユーザーが作成したカードが一覧に含まれていないこと。                                |         |
| **Update** | NOTE形式の更新       | Status 200。DBが更新され、入力に関わらず `question`/`answer` が `null` を維持すること。 |         |
|            | QUIZ形式の更新       | Status 200。DBが更新され、入力に関わらず `content` が `null` を維持すること。           |         |
| **Delete** | カードの削除         | Status 204。DBから対象レコードが削除され、再取得時に取得できないこと。                  |         |

## 3. 異常系・境界値シナリオ

| カテゴリ               | テストケース                   | 確認事項 (Assertion)                                                              | 期待ステータス   |
| :--------------------- | :----------------------------- | :-------------------------------------------------------------------------------- | :--------------- |
| **バリデーション**     | 不正なボディパラメータ         | `deckId` が非数字、`name` が空文字、`type` が定義外 (0, 1以外) の場合。           | 400 Bad Request  |
|                        | 不正なパスパラメータ (PUT)     | `PUT /card/abc` のように `cardId` が非数字の場合、`ParseBigIntIdPipe` が拒否する。 | 400 Bad Request  |
|                        | 不正なパスパラメータ (DELETE)  | `DELETE /card/abc` のように `cardId` が非数字の場合、`ParseBigIntIdPipe` が拒否する。 | 400 Bad Request  |
| **ビジネスロジック**   | 名称の重複                     | 同一ユーザー内で既に存在する `name` を指定して作成・更新を試みた場合。            | 409 Conflict     |
| **認可・セキュリティ** | 未認証アクセス                 | JWTトークンなし、または無効なトークンでリクエストを送った場合。                   | 401 Unauthorized |
|                        | 他人リソースの操作             | 他のユーザーが所有する `cardId` を指定して更新・削除を試みた場合。                | 404 Not Found    |
|                        | 他人デッキへの作成             | 他のユーザーが所有する `deckId` を指定してカード作成を試みた場合。                | 404 Not Found    |
