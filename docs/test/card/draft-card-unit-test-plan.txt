Card試験観点
すべてを網羅せずserviceを重点的に、controllerを薄めに、repositoryはほぼ書かない。

層をまたいだロジックが薄いため結合は行わず、E2Eを実施する。
E2Eにおいて項目は各CRUDの正常と異常に分かれる。
正常はDb更新されて、レスポンスが正常に返ること。
異常はすべての例外は網羅せず一メソッドに一つの例外くらいが投げられるか確認すること
その他として、認可（他人のcardを触れない）、nameの重複、typeの分岐

E2E

正常系
 Create
  正しいuseridとデータで入力して、typeはnote。結果としてdbにCardができて、201が返ること、件数が1件増えていること、返却値とDBの値が一致していること。questionとanswerが入力にかかわらずnullになること
  正しいuseridとデータで入力して、typeはquiz。結果としてdbにCardができて、201が返ること、件数が1件増えていること、返却値とDBの値が一致していること。contentが入力にかかわらずnullになること
  
 Get
  正しいuseridで入力してそのユーザのすべてのCardが取得できて、200が返ること
  あるユーザのCardを取得した際に別UserのCardが混じっていないこと
 Update
  正しいuseridとデータ(cardid含む)で、typeがnote。結果としてCardを更新できて、200が返ること、対象レコードが更新されていること、ほかのレコードに影響がないことquestionとanswerが入力にかかわらずnullになること
  正しいuseridとデータ(cardid含む)で、typeがquiz。結果としてCardを更新できて、200が返ること、対象レコードが更新されていること、ほかのレコードに影響がないことcontentが入力にかかわらずnullになること
 Delete
  正しいuseridとcardidでCardを削除できて、204が返ること、dbから削除されていること

異常系
 Create
  不正な値で入力するとバリデーションエラーが発生し、400が返ること
  すでに存在するcardnameの場合にAlreadyExistExが発生し、409が返ること
 Get
  ログインしていない状態でguardのエラーが出て、401が返ること（認証で行うので不要!）
 Update
  不正な値で入力するとバリデーションエラーが発生し、400が返ること
  すでに存在するcardnameの場合にAlreadyExistExが発生し、409が返ること
  そのユーザに属さないcardidを入力してNotFoundExが発生し、404が返ること
 Delete
  不正な値で入力するとバリデーションエラーが発生し、400が返ること(cardid)
  そのユーザに属さないcardidを入力してNotFoundExが発生し、404が返ること
