未完成なので設計書の概要だけ

## ■要件定義

忘却曲線の沿ったタイミングで問題・メモを表示する暗記サポート機能を提供する。
このサービスのミソはnotionと連携しメモから問題・メモを一括・一部登録できる機能を提供すること。
マルチユーザ対応であること。notion機能だけでなく、他サービスにも拡張できる仕様にすること。

## ■開発手法

- github actions CI (ESLint premitter test build)-
- huskyでcommit前チェック。
- docker bind mountでローカル開発、実行環境とDBはコンテナ
- github issueは機能単位でissueを切って、サブissueはRV容易かつ単体で機能する単位で切る。
- 原則1サブissue=1PR。意味のある作業ごとにcommitする。
- github project kanbanで進捗を管理。

## ■開発思考

前回PJで設計における抽象化の大切さを学んだ。今回は実務を意識したPJということで、
YAGNIとオーバーエンジニアリングのトレードオフを意識した最適な構築を行う。

## ■機能概要

・デッキ機能 カードをまとめる機能。（ノートクイズのどちらも使用できる）
・カード機能 復習単位
ノート形式 単純にノートを表示することで復習に使う。
クイズ形式 裏表のクイズ形式で復習に使う。
・notion連携機能 ノート追加方法の一つとして外部からデータを取得してノートにimportする機能。
・認証認可機能 jwt + passport.jsにより認証認可を行う。
